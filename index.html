88. Merge Sorted Array(Easy)

  • 思路简述：因为两个vector是递增的，并且第一vector过增加了第二个vector的长度。最终想要得到最后的两个归并排序，所以设置三个指针，第一个指向vector1的m长的末尾，第二个指向vector2的n长的末尾，第三个pos指向m+n长的末尾。对比vector1和vector2谁大谁往pos后面填充。注意：如果是填充到最后第二个vector不为空，则第二个vector剩下的元素要添加到第一个pos中剩余的位置里面。

142. Linked List cycle ii(Medium)

  • 思路简述：这里主要使用的思想是Floyd判圈法。关于链表中是否存在环路，通常使用该种方法进行判断。这里采用的是快慢指针的思路。快慢指针都指向开头，慢指针一次走一步，块指针一次走两步。先判断是否存在快慢指针相遇的情况：使用do-while先判断当前fast和fast->next是否为空，为空直接返回nullptr,否则在循环中走各自的步数。如果出现为相遇的情况，说明存在链表回路，此时将fast移动到head处，同时恢复1个步长，当两者再次相遇时，返回fast对应的节点即可。假设慢节点走了n，则快节点走了2n，所以相遇在n点，所以再走n点就可以到达相遇的结点。

操作系统学习记录

发表于 2021-03-09 更新于 2021-04-02

第一讲 操作系统概述

操作系统定义

  • 操作系统是一个控制程序

    • 一个系统软件
    • 控制程序执行过程，防止错误和计算机的不当使用
    • 执行用户程序，给用户程序提供各种服务
    • 方便用户使用计算机系统

  • 操作系统对硬件进行一层隔离，对上面的应用提供我们通常所见到的逻辑这些资源，比如说进程、文件、地址空间

操作系统内核特征：并发、共享、虚拟、异步；

  • 并发：
    • 计算机系统中同时存在多个运行的程序，需要OS管理和调度
  • 共享：
    • 我们多个应用并发进行的时候，宏观上要体现出它们在同时访问资源的情况，微观上要体现它们的互斥访问。
  • 虚拟：
    • 利用多道程序设计技术，让每个用户都觉得有一个计算机专门为他服务，交替的频率非常高，让用户在用的时候感觉不太出来这台机器还有其他用户还在使用。
  • 异步：
    • 程序的执行不是一贯到底，而是走走停停的，向前推进的速度不可预知。只要运行环境相同，OS需要保证程序运行的结果也要相同。

第三讲 启动、中断和系统调用

3.1 BIOS(Basic Input Output System)

  • BIOS位于空闲空间的再往上的一部分，处于640KB~1MB的位置。
  • BIOS提供的基本服务：
    • 基本输入输出的程序
    • 系统设置信息
    • 开机后自检程序
    • 系统自启动程序等
  • BIOS将加载程序从磁盘的引导扇区（512字节）加载到0x7c00，然后跳转到CS:IP = 0000:7c00。
    • 这里加载程序的作用是将操作系统的代码和数据从硬盘加载到内存中，跳转到操作系统的起始位置，控制权交给操作系统
  • 为什么不直接从BIOS里头把操作系统的内核加载进来？
    • 磁盘上有各种各样的文件系统，机器出厂时候不可以直接限制死只使用某一种文件系统。而且为了增加灵活性，在BIOS里头不可能加上认识所有文件系统代码。因此有一个基本的约定，我不需要认识格式也能从硬盘里头读取到第一块，再用加载程序来识别硬盘上的文件系统。
  • BIOS以中断调用的方式提供了基本的I/O功能，在x86系统限制在实模式下工作

SSL/TLS

SSL/TLS的相关定义

​ SSL的英文全称为“Secure Sockets Layer”,中文称作“安全套接层”。1999年，SSL因为广泛的应用，IETF就进行标准化得到TLS（“Trasnport Layer Security”），中文称作“传输层安全协议”。

​ SSL协议位于TCP/IP协议与各种应用层协议之间，为数据通讯提供安全支持。SSL协议可分为两层：

  1. SSL记录协议（SSL Record Protocol）:建立在可靠的传输协议（TCP）上，为高层协议提供数据封装，压缩，加密等基本功能的支持。
  2. SSL握手协议（SSL Handshake Protocol）: 建立在SSL记录协议上，用于在实际的数据传输开始前，通讯双方及进行身份认证，协商加密算法，交换加密密匙等。

SSL/TLS的基本过程

  1. 客户端想服务器索要并验证公共钥匙。
  2. 双方协商生成“对话密钥”
  3. 双发采用“对话密匙”进行通信。

HTTP和HTTPS的部分总结如下：

  1. https协议需要到ca申请证书，一般免费证书较少，因而需要一定费用。
  2. http是超文本传输协议，信息是明文传输，https则是具有安全性的ssl加密传输协议。
  3. http和https使用的是完全不同的连接方式，用的端口也不一样，前者是80，后者是443。
  4. http的连接很简单，是无状态的；HTTPS协议是由SSL+HTTP协议构建的可进行加密传输、身份认证的网络协议，比http协议安全。

Cmake是什么？

​ 首先，C语言如果要生成可执行程序，就需要经过“编译”的这个阶段，不管是什么VisualStudio和CodeBlocks这样子的IDE，在执行绿色三角形的过程中，不管有多少个源文件，IDE自动帮你弄好，只需要执行相应的程序就可以了。

​ 但是，如果是手动执行这个过程的话，最少需要经过“编译”和“链接”这两个过程。“编译”就是处理你的源代码，将其变成一堆二进制文件，“链接”就是可以理解为如果程序中包含多个源代码文件，就要把这些文件的编译结果“组合”起来。

​ 当前，编译器一般一条编译命令也就处理有限的几个文件，如果项目变大了，几千个或者几百个文件，在命令行手写编译命令这就不合适了。因此，弄一个配置文件，描述了编译的操作以及一些参数设置就是很有必要的了，所以makefile就是干这个的。写好makefile，然后永一个叫make的程序，去读这个配置参数，然后进行自动化构建。

​ 但是编译这个操作，一般又和操作系统强相关，你在Linux下编译的操作在Windows下就不一定能用了，所以一般要针对不同平台写不同的makefile。这种麻烦的事情肯定会招人烦，对于跨平台的程序，写一次然后到处运行才是目标，所以Cmake就是这样子。

​ Cmake是跨平台和IDE的一种工具。它定义了一套语言，配套了相应的手册，使用它的语法写一套通用的编译配置的信息（称作CmakeList.txt），然后它会帮你生成各个平台的makefile，然后子就不用写不同的makefile来适应不同的环境了。

​ 总的一句话：

​ CMake是一个构建工具，目标是生成Make-File；Make-File用于指导编译器进行编译
